# ワイヤーフレーム制作プロジェクト

## このプロジェクトについて

このプロジェクトは、WebサイトのワイヤーフレームをFigmaに自動生成するためのプロジェクトです。
クライアントのヒアリングシートを読み込んで、Figma MCPを使ってワイヤーフレームを出力します。

---

## フォルダ構成

```
wireframe-project/
├── CLAUDE.md                          ← このファイル（Claudeへの指示書）
├── README.md                          ← セットアップ手順書（人間向け）
├── .claude/
│   └── settings.json                  ← Claude Codeの設定
├── clients/                           ← クライアントごとのフォルダ
│   └── _template/                     ← 新規クライアント追加時のひな形（触らない！）
│       ├── pre-hearing.md             ← 事前ヒアリングシート
│       ├── hearing.md                 ← 本ヒアリングシート
│       └── output/                    ← 生成したWFのサマリー保存先
├── guidelines/                        ← Claudeが読むルール集
│   ├── wireframe-rules.md             ← WFの品質基準・ルール・文字量基準
│   └── figma-output-guide.md          ← Figmaへの出力手順
└── templates/                         ← ひな形ファイル
    ├── pre-hearing.md                 ← 事前ヒアリングのテンプレート
    └── hearing.md                     ← 本ヒアリングのテンプレート
```

---

## 作業の全体フロー

このプロジェクトでは以下の5ステップで作業を進める。
**必ずこの順番通りに進めること。ステップを飛ばさないこと。**

```
STEP 1：事前ヒアリング読み込み・本ヒアリング先埋め
STEP 2：本ヒアリング完了確認
STEP 3：仮構成・仮テキスト提案
STEP 4：ユーザー確認・修正
STEP 5：Figmaへのワイヤーフレーム出力
```

---

## 各ステップの詳細

### STEP 1：事前ヒアリング読み込み・本ヒアリング先埋め

ユーザーから「pre-hearing.mdを読んでください」と指示されたら以下を行う。

1. 対象クライアントの `clients/クライアント名/pre-hearing.md` を読み込む
2. `guidelines/wireframe-rules.md` を読み込む
3. `guidelines/figma-output-guide.md` を読み込む
4. pre-hearing.mdの内容をもとに、`clients/クライアント名/hearing.md` の埋められる項目を推測して先に記入する
5. 先埋めした項目と、本ヒアリングで確認が必要な残りの項目一覧をユーザーに報告する

### STEP 2：本ヒアリング完了確認

ユーザーから「hearing.mdが埋まりました」と指示されたら以下を行う。

1. `clients/クライアント名/hearing.md` を読み込む
2. 空欄や不明な項目がないか確認する
3. 不明な項目があればユーザーに確認する
4. 全項目が揃っていればSTEP 3に進む

### STEP 3：仮構成・仮テキスト提案

以下の内容をテキストでユーザーに提案する。確認が取れるまでFigmaへの出力は開始しないこと。

**提案する内容：**
- 作成するページの一覧と各ページの目的
- 各ページの主要セクション構成
- 各ページの大見出し（H1）の仮テキスト案
- トップページの各セクションの中見出し（H2）の仮テキスト案
- ヒーローセクションのキャッチコピーとサブコピーの仮テキスト案
- CTAボタンのテキスト案

### STEP 4：ユーザー確認・修正

1. STEP 3の提案内容にOKをもらう
2. 修正依頼があれば修正して再提案する
3. OKをもらったらSTEP 5に進む

### STEP 5：Figmaへのワイヤーフレーム出力

`guidelines/figma-output-guide.md` の手順に従ってFigmaにワイヤーフレームを出力する。
出力完了後は `clients/クライアント名/output/summary.md` にサマリーを保存する。

---

## Claudeへの行動ルール

- hearing.mdに記載のない情報を勝手に追加しないこと
- 不明な点は必ずユーザーに確認すること。勝手に決めないこと
- ページ構成と仮テキストの確認を取らずにFigma出力を開始しないこと
- 各ステップの完了をユーザーに報告してから次のステップに進むこと

---

## 使用ツール

- Figma MCP：ワイヤーフレームのFigma出力に使用

## ショートカット指示一覧

ユーザーから以下の指示が来たとき、対応するアクションを自動で実行すること。

### 「クライアントフォルダを作って、名前は〇〇」

以下を自動で実行する。

1. `clients/_template/` フォルダをまるごとコピーする
2. `clients/〇〇/` という名前で作成する（〇〇はユーザーが指定した名前）
3. 以下の構成になっていることを確認する
   ```
   clients/〇〇/
   ├── pre-hearing.md
   ├── hearing.md
   └── output/
   ```
4. ユーザーに以下を報告する
   ```
   clients/〇〇/ を作成しました！
   次は clients/〇〇/pre-hearing.md を開いて、事前ヒアリングの内容を記入してください。
   記入が終わったら「〇〇のpre-hearingを読んで」と声をかけてください。
   ```

---

### 「〇〇のpre-hearingを読んで」

STEP 1を開始する。

1. `clients/〇〇/pre-hearing.md` を読み込む
2. `guidelines/wireframe-rules.md` を読み込む
3. `guidelines/figma-output-guide.md` を読み込む
4. pre-hearing.mdの内容をもとに `clients/〇〇/hearing.md` の埋められる項目を先に記入する
5. 以下をユーザーに報告する
   - 先埋めした項目の一覧
   - 本ヒアリングで確認が必要な残りの項目一覧

---

### 「hearing埋まったよ」

STEP 2→3を実行する。

1. `clients/〇〇/hearing.md` を読み込む（〇〇は直前に作業していたクライアント名）
2. 空欄や不明な項目がないか確認する
3. 不明な項目があればユーザーに確認する
4. 全項目が揃っていれば、以下をテキストでユーザーに提案する（STEP 3）
   - 作成するページの一覧と各ページの目的
   - 各ページの主要セクション構成
   - 各ページの大見出し（H1）の仮テキスト案
   - トップページの各セクションの中見出し（H2）の仮テキスト案
   - ヒーローセクションのキャッチコピーとサブコピーの仮テキスト案
   - CTAボタンのテキスト案

---

### 「OKです」または「修正して〇〇に変えて」

STEP 4を実行する。

- 「OKです」の場合 → STEP 5（Figma出力）を開始する
- 「修正して〇〇に変えて」の場合 → 指定箇所を修正して再提案する。再度OKをもらってからSTEP 5に進む

---

### 「どこまで進んでる？」

現在の作業状況を以下の形式で報告する。

```
【現在の進捗】
対象クライアント：〇〇
現在のステップ：STEP 〇（〇〇の作業中）
完了済み：STEP 1、STEP 2
残り：STEP 3、STEP 4、STEP 5
```
